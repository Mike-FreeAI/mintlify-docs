# Global Settings

The `mint.json` file is the central configuration file for your Mintlify documentation site. This guide walks through all available configuration options to help you customize your documentation.

## Core Settings

These fundamental settings define the basic identity and appearance of your documentation.

### Project Information

<ResponseField name="name" type="string" required>
- Used as the global title for your documentation
- Appears in browser tabs and search results
- Example: `mintlify`
</ResponseField>

### Visual Identity

<ResponseField name="logo" type="string or object">
Configure your documentation's logo display:

- **Simple Usage**: Provide a direct path to your logo image
- **Advanced Usage**: Configure separate logos for light/dark modes:
  <Expandable title="Logo Options">
    <ResponseField name="light" type="string">Path to light mode logo</ResponseField>
    <ResponseField name="dark" type="string">Path to dark mode logo</ResponseField>
    <ResponseField name="href" type="string" default="/">Logo click destination</ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="favicon" type="string">
Path to your site's favicon image
</ResponseField>

### Theme Colors

<ResponseField name="colors" type="Colors">
Define your documentation's color scheme:

<Expandable title="Color Configuration">
    <ResponseField name="primary" type="string" required>
    - Main brand color
    - Used for highlights and accents
    - Applied to headers and interactive elements
    </ResponseField>
    <ResponseField name="light" type="string">
    - Alternate color for dark mode
    - Used for accents in dark mode
    </ResponseField>
    <ResponseField name="dark" type="string">
    - Color for important buttons
    - Used for high-contrast elements
    </ResponseField>
    <ResponseField name="background" type="object">
      <Expandable title="Background Colors">
        <ResponseField name="light" type="string" required>Light mode background</ResponseField>
        <ResponseField name="dark" type="string" required>Dark mode background</ResponseField>
      </Expandable>
    </ResponseField>
</Expandable>
</ResponseField>

## Navigation Structure

### Main Navigation

<ResponseField name="navigation" type="Navigation[]" required>
Define your documentation's page structure:
  <Expandable title="Navigation Structure">
    <ResponseField name="group" type="string">
    - Name of the navigation group
    - Example: `Getting Started`
    </ResponseField>
    <ResponseField name="pages" type="string[]">
    - List of page paths
    - Example: `["quickstart", "installation", "overview"]`
    </ResponseField>
  </Expandable>
</ResponseField>

### Top Bar Elements

<ResponseField name="topbarLinks" type="TopbarLink[]">
Add links to your top navigation bar:
  <Expandable title="TopbarLink Structure">
    <ResponseField name="name" type="string">Link text</ResponseField>
    <ResponseField name="url" type="string">Link destination</ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="topbarCtaButton" type="Call to Action">
Configure a prominent call-to-action button:
  <Expandable title="CTA Options">
    <ResponseField name="type" type={'"link" or "github"'} default="link">Button type</ResponseField>
    <ResponseField name="url" type="string">Button destination</ResponseField>
    <ResponseField name="name" type="string">Button text (for link type)</ResponseField>
  </Expandable>
</ResponseField>

## Advanced Navigation

### Version Control

<ResponseField name="versions" type="string[]">
Enable documentation versioning:
- Add version names to create a version selector
- Only needed for multi-version documentation
</ResponseField>

### Anchor Navigation

<ResponseField name="anchors" type="Anchor[]">
Configure sidebar anchor points:
  <Expandable title="Anchor Configuration">
    <ResponseField name="icon" type="string">Font Awesome icon name</ResponseField>
    <ResponseField name="name" type="string">Anchor label text</ResponseField>
    <ResponseField name="url" type="string">Section identifier</ResponseField>
    <ResponseField name="color" type="string">Icon background color</ResponseField>
    <ResponseField name="version" type="string">Version visibility control</ResponseField>
    <ResponseField name="isDefaultHidden" type="boolean" default="false">Initial visibility state</ResponseField>
    <ResponseField name="iconType" default="duotone" type="string">Icon style variant</ResponseField>
  </Expandable>
</ResponseField>

## API Documentation

### OpenAPI Configuration

<ResponseField name="openapi" type="string | string[]">
Define OpenAPI specification sources:

<CodeGroup>
```json Single Source
"openapi": "https://example.com/openapi.json"
```

```json Multiple Sources
"openapi": [
  "https://example.com/openapi1.json",
  "/openapi2.json"
]
```
</CodeGroup>
</ResponseField>

### API Settings

<ResponseField name="api" type="API">
Configure API playground behavior:
  <Expandable title="API Settings">
    <ResponseField name="baseUrl" type="string">Base endpoint URL</ResponseField>
    <ResponseField name="auth" type="Auth">Authentication configuration</ResponseField>
    <ResponseField name="playground" type="Playground">Interactive playground settings</ResponseField>
    <ResponseField name="maintainOrder" type="boolean">Parameter order preservation</ResponseField>
  </Expandable>
</ResponseField>

## Additional Features

### Social Integration

<ResponseField name="footerSocials" type="FooterSocials">
Add social media links to your documentation footer:
```json
{
  "x": "https://x.com/mintlify",
  "github": "https://github.com/mintlify"
}
```
</ResponseField>

### User Feedback

<ResponseField name="feedback" type="Feedback">
Enable user feedback mechanisms:
  <Expandable title="Feedback Options">
    <ResponseField name="suggestEdit" type="boolean" default="false">Enable edit suggestions</ResponseField>
    <ResponseField name="raiseIssue" type="boolean" default="false">Enable issue reporting</ResponseField>
  </Expandable>
</ResponseField>

### Display Settings

<ResponseField name="modeToggle" type="ModeToggle">
Customize theme switching options:
  <Expandable title="Mode Toggle Options">
    <ResponseField name="default" type={'"light" or "dark"'}>Default theme setting</ResponseField>
    <ResponseField name="isHidden" type="boolean" default="false">Toggle visibility control</ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="backgroundImage" type="string">
Add a background image to all pages in your documentation
</ResponseField>
